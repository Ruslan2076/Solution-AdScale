# Задание 2: Проектирование RTB и интеграции с DSP

Детальный дизайн Bidding & Delivery Service, критичного компонента AdScale, выделенного из монолита первым по ADR-002 (задание 1). Документы описывают, как сервис соблюдает требование DSP-партнёра по времени отклика не более 80 ms и остаётся надёжным при частичных отказах зависимых сервисов.

## Структура документов

- **bidding-service.md**: спецификация сервиса ставок, границы, внешний и внутренний API, зависимости от других сервисов, модель данных.
- **interaction.md**: протоколы взаимодействия с обоснованием, gRPC для внутренних вызовов, REST/OpenRTB для DSP-партнёра, Kafka для потоковой обработки событий.
- **api-gateway.md**: дизайн API Gateway для DSP-интеграции, маршрутизация, аутентификация, rate limiting, мониторинг времени отклика.
- **reliability-patterns.md**: паттерны надёжности, Circuit Breaker для вызовов зависимых сервисов, Retry с экспоненциальной задержкой, идемпотентность финансовых операций, резервные стратегии.
- **diagrams/bid-request-sequence.puml**: sequence-диаграмма полного цикла обработки bid request, синхронный hot path и асинхронная публикация событий.
- **diagrams/bidding-service-component.puml**: component-диаграмма внутреннего устройства Bidding & Delivery Service.

## Ключевое архитектурное решение

У Bidding & Delivery Service нет ни одной синхронной зависимости от других доменных сервисов на критичном пути. Все взаимодействия с Advertiser Dashboard Service и Financial Service реализованы асинхронно через Kafka: сервис работает с локальной read-моделью и кэшированным остатком бюджета, обновляемыми потоком событий, а не синхронными вызовами.

Это решение напрямую следует из требований надёжности: зависимость, способная замедлиться или отказать, не может находиться в пути с бюджетом задержки 80 ms без риска нарушить контрактный SLA перед DSP-партнёром. Компромиссом является временная неточность данных о кампаниях и бюджете в пределах окна отставания синхронизации, которая компенсируется идемпотентной обработкой финансовых операций на стороне Financial Service.
